This week at DBC we really began to dive into using classes in Ruby. I think this was really new to most of us but is looking like an incredibly useful tool. First let's talk about what classes are. Since everything in Ruby is pretty much an object, we can think of classes as the blueprints from which these objects are created. If we wanted to get technical we could say that an object is an instance of a class. An easy way to think about this is to say your car is an instance of the class cars. Most cars have the same characteristics but differ on the value of these things. Size of the engine and fuel capactiy for example. These attributes are shared between the cars but their exact number may be different.

The next section of the code you will notice is where we are creating instance variables. An instance variable begins with a "@" symbol and is available anywhere in the class like in the favorite method I create after. This method does not have access to the local variable "title" but it can use the instance variable "@title". Any other methods we create inside the class can also use these instance variables.
