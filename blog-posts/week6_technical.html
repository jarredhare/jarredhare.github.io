What is Ruby inheritance?

When creating a new class in Ruby you are able to indicate that this new class will get most or all of its features from a parent class. The child class is said to inherit traits from the parent such as general methods that you would like to apply to both. There are several ways this can be done and I will show them here with the example of cars. Since I drive a Jetta I will define a child class called Jetta that inherits traits from a parent class called Car. Simple enough. The most straightforward form of inheritance is when a method is defined in the parent but not in the child class like the method "type". An instance of the jetta class can still access this method even though it is not defined within it's class specifically.

Since Jetta had no type method defined inside, it just used the one inherited from the Car class. But we can also explicitly define the type method within the Jetta class and override the features inherited from Car. You can see that in this case the jetta object uses the method defined within the Jetta class. One final way that you can interact with the inheritance is the super() function that is built into Ruby. Super will get the parent class version of the method and run it when called. You can see the output here. Inheritance is generally used when there is clearly related repeated code that you can combine and distribute to all of the related classes.

What is composition?

Composition is another way to accomplish the exact same thing as inheritance. Like most things in Ruby, there is always more than one way to solve a problem. Composition actually uses another class or module in the current class as opposed to using implicit inheritance. It is most easily seen with an example here using the same classes as before. The only difference is that in this first example we need to create an instance of the car class in our initialize method and go from there.

This final quick example shows how to once again accomplish the same thing using a module. By including the module in the class the jetta object has access to the methods defined in the module. Modules work great in cases where you want to combine and save code to be used in a lot of unrelated places.
